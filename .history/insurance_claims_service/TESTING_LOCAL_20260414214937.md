# Local Development & Testing Guide

This guide provides detailed instructions for setting up and testing the Insurance Claims API on your local machine.

## Prerequisites

### Required Software

1. **Python 3.11+**
   - macOS: `brew install python@3.11`
   - Ubuntu/Debian: `sudo apt-get install python3.11`

2. **PostgreSQL 15+** (Choose one option)
   - **Option A - Local Installation:**
     - macOS: `brew install postgresql@15`
     - Ubuntu/Debian: `sudo apt-get install postgresql-15`
   
   - **Option B - Docker (Recommended):**
     ```bash
     docker-compose up -d postgres
     ```

3. **Redis 7+** (Choose one option)
   - **Option A - Local Installation:**
     - macOS: `brew install redis`
     - Ubuntu/Debian: `sudo apt-get install redis`
   
   - **Option B - Docker (Recommended):**
     ```bash
     docker-compose up -d redis
     ```

### Optional Tools

- **Git**: For version control
- **Docker**: For running PostgreSQL/Redis containers
- **Visual Studio Code**: Recommended IDE
- **Postman**: For API testing

### 2. Subsequent Runs (Quick Start)

After initial setup, use the quick start script:


## Manual Setup (Step-by-Step)

If you prefer manual setup or the automated scripts fail:

### Step 1: Clone Repository

```bash
git clone <repository-url>
cd insurance_claims/insurance_claims_service
```

### Step 2: Create Virtual Environment

```bash
# Create venv
python3 -m venv venv

# Activate
source venv/bin/activate
```

### Step 3: Install Dependencies

```bash
# Upgrade pip
pip install --upgrade pip

# Install production dependencies
pip install -r requirements.txt

# Install development dependencies
pip install -r requirements-dev.txt
```

### Step 4: Configure Environment

```bash
# Copy example environment file
cp .env.example .env

# Edit .env file - IMPORTANT: Set these variables
# - SECRET_KEY (generate with: openssl rand -hex 32)
# - DATABASE_URL
# - REDIS_URL
```

**Example .env configuration:**
```env
# Application
APP_NAME=Insurance Claims API
DEBUG=True
ENVIRONMENT=development

# Database
DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/insurance_claims

# Redis
REDIS_URL=redis://localhost:6379/0

# Security (generate with: openssl rand -hex 32)
SECRET_KEY=your-secret-key-here-min-32-chars
```

### Step 5: Start Required Services

**Option A - Using Docker (Recommended):**
```bash
# Start PostgreSQL and Redis
docker-compose up -d postgres redis

# Verify services are running
docker-compose ps

# View logs
docker-compose logs -f postgres redis
```

**Option B - Local Services:**
```bash
# Start PostgreSQL (macOS)
brew services start postgresql@15

# Start Redis (macOS)
brew services start redis

# Verify PostgreSQL
psql -h localhost -U postgres -c "SELECT version();"

# Verify Redis
redis-cli ping
```

### Step 6: Initialize Database

```bash
# Create database (if using local PostgreSQL)
createdb insurance_claims

# Run migrations
alembic upgrade head
```

### Step 7: Run Application

```bash
# Development mode (with auto-reload)
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# The application will be available at:
# - API: http://localhost:8000
# - Swagger UI: http://localhost:8000/docs
# - ReDoc: http://localhost:8000/redoc
```


## Troubleshooting

### Port Already in Use

**Problem:** Error "Address already in use" on port 8000

**Solution:**
```bash
# Find process using port 8000
lsof -i :8000

# Kill the process
kill -9 <PID>
```

### Database Connection Failed

**Problem:** Cannot connect to PostgreSQL

**Solutions:**

1. **Check if PostgreSQL is running:**
   ```bash
   # Docker:
   docker-compose ps
   
   # Local (macOS):
   brew services list
   
   # Local (Linux):
   sudo systemctl status postgresql
   ```

2. **Verify connection details in .env:**
   ```env
   DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/insurance_claims
   ```

3. **Test connection manually:**
   ```bash
   psql -h localhost -U postgres -d postgres
   ```

4. **Reset Docker database:**
   ```bash
   docker-compose down -v
   docker-compose up -d postgres
   ```

### Redis Connection Failed

**Problem:** Cannot connect to Redis

**Solutions:**

1. **Check if Redis is running:**
   ```bash
   # Docker:
   docker-compose ps
   
   # Local:
   redis-cli ping
   ```

2. **Start Redis:**
   ```bash
   # Docker:
   docker-compose up -d redis
   
   # Local (macOS):
   brew services start redis
   ```

### Import Errors

**Problem:** ModuleNotFoundError or ImportError

**Solutions:**

1. **Ensure virtual environment is activated:**
   ```bash
   # You should see (venv) in your prompt
   source venv/bin/activate
   ```

2. **Reinstall dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Check Python version:**
   ```bash
   python --version  # Should be 3.11+
   ```

### Alembic Errors

**Problem:** Alembic migration fails

**Solutions:**

1. **Check database connection:**
   ```bash
   alembic current
   ```

2. **Verify alembic.ini configuration:**
   ```ini
   sqlalchemy.url = postgresql+asyncpg://postgres:postgres@localhost:5432/insurance_claims
   ```

3. **Reset migrations (development only):**
   ```bash
   # Drop all tables
   alembic downgrade base
   
   # Re-apply migrations
   alembic upgrade head
   ```

## Development Tips

1. **Use environment variables for configuration** - Never hardcode credentials
2. **Write tests for new features** - Maintain test coverage above 80%
3. **Use type hints** - Helps with IDE autocomplete and catches errors early
4. **Document API endpoints** - Add docstrings to all endpoint functions
5. **Use async/await** - Leverage FastAPI's async capabilities
6. **Check logs regularly** - Monitor application output for errors
7. **Profile slow endpoints** - Use profiling tools to identify bottlenecks
8. **Keep dependencies updated** - Regularly update requirements.txt
